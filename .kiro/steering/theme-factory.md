# theme-factory — 文档主题与排版规范

> 本文件等价于 Claude 的 `/theme-factory` skill，在本仓库 (priceAction) 内自动生效。
> 用于统一所有交易研究文档的视觉风格、标题层级、强调规则。

## 1. 标题层级

```
# 一级 — 仅用于文档标题（每文件唯一）
## 二级 — 主章节（一、二、三…）
### 三级 — 子章节（1.1, 1.2 或 步骤①②③）
#### 四级 — 极少使用，宁可用粗体段落代替
```

- 二级标题前缀使用中文数字编号：`## 一、系统目标与边界`。
- 三级标题前缀使用阿拉伯数字：`### 1.1 主过滤`。

## 2. 强调规则

| 用法 | 写法 | 含义 |
|---|---|---|
| **粗体** | `**强趋势**` | 业务关键概念首次出现 |
| `代码体` | `EMA20` | 变量、函数、参数名 |
| *斜体* | 不使用 | 中文斜体可读性差，禁用 |
| > 引用 | `> 关键规则：…` | 提示、警告、注释 |

## 3. 列表

- 条件列表用 `+`（不要混用 `-` 和 `*`）。
- 步骤列表用阿拉伯数字 `1.` `2.`。
- 嵌套层级最多两层。

## 4. 表格

强制：

- 第一列为分类/名称，最后一列为说明/备注。
- 表格前必须有一句话引导（不要孤立的表格）。
- 阈值类参数表必须包含列：`参数 | 默认值 | 含义 | 备注`。

## 5. 代码块

- 伪代码用 ` ```plain `（GitHub 不会高亮，避免误导）。
- Pine Script / MQL5 / Python 用对应语言标签。
- 代码块前用一行小标题（粗体或四级标题）说明用途。

## 6. 元信息块

每份**版本化文档**首页必须有：

```markdown
**版本：v{X.Y} | 日期：{YYYY-MM} | 品种：{ES} | 周期：{5min} | 状态：{草稿/生效/废弃}**
```

## 7. 章节顺序约定

设计文档章节固定顺序（不要打乱）：

1. 系统目标与边界
2. 决策流程总览
3. 每个步骤的详细规则（按编号）
4. 核心参数一览（表格）
5. 风险预算与仓位管理
6. 执行假设
7. 已知缺口与迭代方向
8. 附录（变量字典、术语表）

## 8. 颜色与符号

继承 `canvas-design.md` 的符号语义（✅ ❌ ⚠️ 📌 🔁 🛑）。除此之外不使用装饰性图标。

## 9. 链接与引用

- 文档内跨章节引用：`参见步骤 7`，不写"上文"、"下文"等模糊指代。
- 引用外部资源使用脚注或末尾参考文献节，不在正文中夹杂裸 URL。

## 10. 中英混排

- 中英文之间留一个半角空格：`使用 EMA20 判定` 而非 `使用EMA20判定`。
- 中文数字、英文术语、阿拉伯数字之间同样留空格：`12 根 K 线`。
- 标点符号：中文段落用全角标点（，。；：），代码注释和英文段落用半角。
