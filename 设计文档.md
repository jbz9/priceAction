# ES 5分钟 H2 多头交易系统
**版本：v1.0 | 日期：2026年5月 | 交易品种：ES（E-mini S&P 500 期货）**

---

## 一、系统目标与边界
本系统专注于 ES 五分钟图上的**高质量 H2 多头机会**。

核心原则：

+ **不追求捕捉所有上涨**，只在强多头趋势中的健康回调后，等待第二次多头恢复尝试
+ **过滤低质量交易**，宁可错过，不可随意入场
+ **首期只做多头**，L2 空头与区间边缘失败突破不在本文档内

硬性约束：

+ 交易品种：ES（E-mini S&P 500 期货）
+ 交易时间：美盘时段
+ 主交易周期：5分钟
+ 所有判断**只使用已收盘K线**，当前未收盘K线不参与信号确认

---

## 二、完整决策流程（10步序列）
一笔交易必须依次通过以下10步。任意一步不通过，系统返回等待状态。

```plain
步骤 1 → 1小时背景过滤
步骤 2 → 5分钟强多头趋势判断（5项条件）
步骤 3 → 健康回调判断（3项条件）
步骤 4 → 识别合理H1（4项条件）
步骤 5 → H1结果判断（成功则不等H2）
步骤 6 → 识别合理H2（5项条件，H1弱或失败后）
步骤 7 → 入场前风险检查（3项条件）
步骤 8 → 挂买入止损单（仅下一根K有效）
步骤 9 → 入场K线与跟随K线质量评估
步骤 10 → 止损、止盈与移动止损管理
```

---

## 三、步骤1：1小时背景过滤
**目的：** 防止5分钟出现局部强趋势，但1小时仍处于交易区间或衰竭趋势，导致系统误判。

### 主过滤（必须通过）
+ ✅ 5min EMA20 smoothed_slope > 0.15（见步骤2定义）
+ ✅ 1H EMA20 slope > 0（方向过滤，1小时多头方向向上）

### 辅助过滤（必须通过）
+ ✅ 1H 最近回调幅度 < 前段上涨的 50%
+ ✅ 1H 不处于大交易区间中段

```plain
大交易区间定义：
  区间宽度 > 3 × ATR14（1H）
  且当前价格在区间 30%–70% 位置
  → 满足以上两项 = 区间中段，不做
```

---

## 四、步骤2：5分钟强多头趋势（5项条件，须同时成立）
### 条件① EMA20方向向上
```plain
slope(i)        = (EMA20[i] - EMA20[i+5]) / ATR14[i]
smoothed_slope  = average(slope[1], slope[2], slope[3])

Direction_Up_5m = smoothed_slope > 0.15
```

> 注：全部使用已收盘K线（i=1为最近已收盘K线），避免使用当前未收盘bar。
>

---

### 条件② 价格持续接受在EMA20上方
```plain
Price_Acceptance_Above_EMA20 =
  近12根已收盘K线中，收盘价 > EMA20 的数量 >= 8
  AND 近12根内不出现连续3根收盘 < EMA20
```

---

### 条件③ 多头K线质量合格
```plain
Bull_Close(i)      = close[i] > open[i]

Strong_Bull_Bar(i) =
  close[i] > open[i]
  AND close[i] >= low[i] + 0.70 × range[i]   ← 收盘在上方30%区域
  AND body[i] >= 0.40 × ATR14[i]              ← 实体不小于0.4ATR

Bull_Bar_Quality_OK =
  近12根已收盘K线中：
    Bull_Close 数量 >= 7
    AND Strong_Bull_Bar 数量 >= 3
```

---

### 条件④ 摆动结构偏多
**有效摆动点定义（搜索窗口：最近50根已收盘K线）：**

```plain
有效摆动高点(i) =
  high[i] > high[i-1] AND high[i] > high[i-2]
  AND high[i] > high[i+1] AND high[i] > high[i+2]   ← 2-bar pivot
  AND 与前一有效摆动低点距离 >= 0.75 × ATR14
  AND 与前一摆动点间隔 >= 3根K线

有效摆动低点(i) =
  low[i] < low[i-1] AND low[i] < low[i-2]
  AND low[i] < low[i+1] AND low[i] < low[i+2]
  AND 与前一有效摆动高点距离 >= 0.75 × ATR14
  AND 与前一摆动点间隔 >= 3根K线

主要摆动低点(i) =
  是有效摆动低点
  AND 其后20根K线内最高点 - low[i] >= 1.25 × ATR14
  ← 只有真正引发有效上涨的低点，才视为主要多头防守低点
```

**结构条件：**

```plain
Higher_High_OK =
  最近有效摆动高点 > 前一有效摆动高点 + 0.25 × ATR14

No_Clear_Break_Of_MajorLow =
  最近有效摆动低点 >= 前一主要摆动低点 - 0.25 × ATR14

Swing_Structure_Long_OK = Higher_High_OK AND No_Clear_Break_Of_MajorLow
```

---

### 条件⑤ 无明显空头反转压力
```plain
Strong_Bear_Bar(i) =
  close[i] < open[i]
  AND close[i] <= low[i] + 0.30 × range[i]   ← 收盘在下方30%区域
  AND body[i] >= 0.40 × ATR14[i]

No_Bear_Reversal_Pressure =
  近5根已收盘K线中，不出现连续3根 Strong_Bear_Bar
  AND 若出现连续3根强空头K，且最新一根收于EMA20下方 → 强多头趋势立即失效
```

---

## 五、步骤3：健康回调（3项条件，须同时满足）
强趋势成立后，不直接寻找入场，须等待健康回调。

**关键变量定义：**

```plain
Leg_Start      = 最近主要摆动低点（last 50 bars）
Leg_High       = Leg_Start之后的最高点
Current_PB_Low = 从Leg_High到当前已收盘K线的最低点
Pullback_Ratio = (Leg_High - Current_PB_Low) / (Leg_High - Leg_Start)
```

**三项条件：**

```plain
① Pullback_Ratio < 0.50                   ← 回调深度 < 上涨腿50%
② Current_PB_Low > Leg_Start              ← 回调低点高于上涨腿起点
③ 当前回调中不出现连续3根 Strong_Bear_Bar  ← 无极端空头压力
```

> **失效条件（任一）：** 回调超过50%、跌破Leg_Start、出现连续3根强空头K  
→ 系统停止寻找H2，返回等待
>

---

## 六、步骤4：识别合理H1（4项条件，须同时满足）
H1 = 健康回调后第一次突破前一根K线高点的向上尝试。

**条件①：有效回调已存在**

```plain
回调 >= 2根已收盘K线
AND 回调深度 >= 0.25 × ATR14
AND 回调深度 < Leg_High的50%（Pullback_Ratio < 0.50）
```

**条件②：H1信号K质量合格**

```plain
H1信号K 非 Strong_Bear_Bar
AND H1收盘 >= low[H1] + 0.40 × range[H1]   ← 收盘不能太差（下方40%以上）
```

**条件③：非高位噪音**

```plain
不同时满足以下两项（满足两项则视为噪音H1，不合格）：
  H1收盘接近Leg_High（距离 < 0.25 × ATR14）
  AND 回调深度 < 0.50 × ATR14
```

**条件④：无近期强空头压制**

```plain
近3根已收盘K线中 Strong_Bear_Bar 数量 < 2
AND 近5根已收盘K线中不出现：连续2根Strong_Bear_Bar 且第2根收于EMA20下方
```

---

## 七、步骤5：H1结果判断
H1触发后，不立刻找H2，先判断H1结果。

**H1成功（不等H2）：**

```plain
H1触发后2根已收盘K线内出现 Strong_Follow_Through
OR 价格直接收盘突破 Leg_High
→ H1成功，趋势已直接恢复，本轮不再等待H2
```

**H1弱或失败（才找H2）：**

```plain
H1触发后2根K线内：
  无 Strong_Follow_Through
  且未突破 Leg_High
  且价格再次回调
→ H1弱或失败，系统进入步骤6寻找H2
```

> **关键规则：** H1本身不合理（步骤4未通过）→ H2不成立，系统返回等待。
>

---

## 八、步骤6：识别合理H2（5项条件，须同时满足）
H2 = 合理H1弱或失败后，第二次突破前一根K线高点的向上尝试。

**条件① 前提成立**

```plain
合理H1已存在 AND H1弱或失败
```

**条件② 仍在健康回调内**

```plain
Pullback_Ratio < 0.50
AND Current_PB_Low > Leg_Start
```

**条件③ 时间限制**

```plain
从Leg_High开始计算，回调持续K线数 <= 12根
→ 超过12根，趋势可能转入交易区间，停止寻找强趋势H2
```

**条件④ H2信号K质量（须优于H1）**

```plain
H2信号K 为多头收盘（close > open）
AND 非 Strong_Bear_Bar
AND close[H2] >= low[H2] + 0.40 × range[H2]   ← 收盘在上方40%区域
```

**条件⑤ 入场触发方式**

```plain
Entry_Price = high[H2_bar] + 1 tick
入场方式：买入止损单（Buy Stop）
有效期：H2信号K收盘后的下一根K线
→ 若下一根K结束仍未触发：撤单，信号失效，不追入
```

---

## 九、步骤7：入场前风险检查（3项条件）
合理H2成立后，挂单前必须通过风险检查。

**计算定义：**

```plain
Entry_Price    = high[H2_bar] + 1 tick
Initial_Stop   = low[H2_bar] - 1 tick
Stop_Distance  = Entry_Price - Initial_Stop
Risk_R         = Stop_Distance
```

**条件① 止损上限**

```plain
Stop_Distance <= 1.5 × ATR14
→ 不满足：取消本次交易
```

**条件② 止损下限**

```plain
Stop_Distance >= 0.35 × ATR14
→ 不满足：取消交易，或将止损改为 Current_PB_Low - 1tick 后重新计算
```

**条件③ 结构止损合理**

```plain
Initial_Stop <= Current_PB_Low
OR abs(Initial_Stop - Current_PB_Low) <= 0.25 × ATR14
→ 若H2低点远高于回调低点，结构保护不足，取消交易
```

**止盈空间检查（决定仓位模式）：**

```plain
Scalp_Target          = Entry_Price + 1.0 × Risk_R
Swing_Target_Primary  = Leg_High
Swing_Target_Extended = Current_PB_Low + (Leg_High - Leg_Start)   ← 等幅延伸

Leg_High_Too_Close = (Leg_High - Entry_Price) < 0.5 × ATR14
Swing_RR_OK        = (Swing_Target_Primary - Entry_Price) / Risk_R >= 1.5

IF Leg_High_Too_Close OR NOT Swing_RR_OK:
    Trade_Mode = Scalp_Only       ← 100%仓位在Scalp_Target止盈
ELSE:
    Trade_Mode = Scalp_Plus_Swing ← 50%仓位Scalp，50%仓位Swing
```

---

## 十、步骤8：挂买入止损单
```plain
挂单：Buy Stop @ Entry_Price
有效期：H2信号K收盘后的下一根K线

若下一根K线结束仍未触发：
  → 撤单
  → 信号失效
  → 不等待，不追入，返回等待状态
```

> **逻辑依据：** 高质量H2应快速吸引买盘。无法在下一根K突破H2高点，说明多头动能不足。
>

---

## 十一、步骤9：入场K线与跟随K线评估
### 入场K线（Entry Bar）= H2之后触发买入止损单的K线
**质量分级：**

```plain
强入场K（Strong_Entry_Bar）：
  close > open
  AND close >= low + 0.70 × range   ← 收盘在上方30%区域
  AND close > Entry_Price
  AND range <= 1.5 × ATR14

合格入场K（Entry_Bar_Quality_OK）：
  range <= 2.0 × ATR14
  AND low <= EMA20 + 1.5 × ATR14   ← 非超涨离均线过远
  AND close >= low + 0.30 × range  ← 收盘非极弱

弱入场K（Weak_Entry_Bar）：
  已触发 AND Entry_Bar_Quality_OK = FALSE
  OR close < Entry_Price
  OR close < low + 0.50 × range
```

### 跟随K线（Follow Bar）= 入场K线之后第一根已收盘K线
**质量分级：**

```plain
强跟随（Strong_Follow_Through）：
  close > Entry_Price
  AND close > open
  AND close >= low + 0.60 × range
  AND close >= close[Entry_Bar]

弱跟随（Weak_Follow_Through）：
  close > Entry_Price
  AND（收盘位置弱 OR 空头收 OR close < close[Entry_Bar]）

失败跟随（Failed_Follow_Through）：
  close <= Entry_Price
  OR low < low[H2_bar]
  OR 是 Strong_Bear_Bar
```

**根据评估结果的行动：**

| 组合 | 模式 | 操作 |
| --- | --- | --- |
| 强入场K + 强跟随 | 正常持仓 | 进入步骤10全面管理 |
| 合格入场K + 弱跟随 | 防守模式 | 不加仓，考虑快速保本或缩短目标 |
| 弱入场K / 失败跟随 | 低质量 | 若未成交撤单；若已成交立即防守退出 |

## 十二、步骤10：止损与止盈管理
### 仓位分配（由步骤7决定）
```plain
Scalp_Plus_Swing：50% Scalp + 50% Swing
Scalp_Only：      100% Scalp
```

### Scalp仓位出场
```plain
价格触及 Scalp_Target（= Entry_Price + 1R）→ 全平Scalp仓位
```

### Swing仓位移动止损（三阶段）
**阶段一：入场确认阶段**

```plain
触发条件：入场后出现 Strong_Follow_Through
操作：Swing止损上移至 Entry_Price - 1 tick（保本）
```

**阶段二：突破前高阶段**

```plain
触发条件：价格收盘突破 Leg_High
操作：Swing止损 = max(
    Entry_Price - 1 tick,
    Leg_High - 0.25 × ATR14,
    最近2根K线低点 - 1 tick
)
```

**阶段三：趋势持续阶段**

```plain
触发条件：价格继续上涨
操作：Swing止损 = max(
    当前Swing止损,
    近10根K线内最近有效摆动低点 - 1 tick
)

到达 Swing_Target_Extended → 全平Swing仓位
```

### 防守退出（任一条件触发 → 全平剩余仓位）
```plain
① Failed_Follow_Through（跟随K失败）
② 收盘跌破 Current_PB_Low（回调低点被有效跌破）
③ 收盘跌破 EMA20 且 出现 Strong_Bear_Bar
④ 入场后6根已收盘K线内，浮盈未达 +0.5 × Risk_R
```

## 十三、交易结束条件
一笔交易在以下任一情况下结束：

1. 未触发入场而撤单
2. 步骤7入场前风险检查不通过
3. 触发Initial_Stop（初始止损）
4. Scalp_Only模式：到达Scalp_Target全平
5. Scalp_Plus_Swing模式：Scalp仓位在1R止盈，Swing仓位到达延伸目标
6. 触发防守退出（步骤10中任一条件）
7. 触发跟踪止损（阶段三移动止损被触及）

## 十四、核心参数一览
| 参数 | 定义 | 备注 |
| --- | --- | --- |
| EMA周期 | 20 | 5分钟图 |
| ATR周期 | 14 | 5分钟图，用于标准化所有距离 |
| slope平滑 | 3根K平均 | 防止斜率噪音 |
| slope阈值 | 0.15 | 强趋势判定 |
| 价格接受窗口 | 12根K | 条件② |
| 多头K质量窗口 | 12根K | 条件③ |
| 摆动点搜索窗口 | 50根K | 条件④ |
| 空头反转压力窗口 | 5根K | 条件⑤ |
| 最大回调深度 | 50% | 健康回调上限 |
| 最大回调时间 | 12根K | H2有效期 |
| 止损上限 | 1.5 × ATR14 | 风险上限 |
| 止损下限 | 0.35 × ATR14 | 防噪音扫损 |
| Swing RR要求 | 1.5R | 进入Swing模式的最低要求 |
| 动能失速判定 | 6根K内未达+0.5R | 防守退出条件④ |

## 十五、已知缺口与后续迭代方向
> 以下为v1.0的已知局限，供后续版本完善参考：
>

1. **L2空头逻辑**：本版本未包含，后续在特定1H背景下补充
2. **区间边缘失败突破（Failed Breakout）**：第二种模式，待单独建立
3. **盘前/盘后过滤**：美盘时段内是否需要排除开盘前30分钟或特定事件窗口，待验证
4. **1小时背景的动态更新频率**：5分钟图每根K收盘时是否需要重新检查1H背景，逻辑未完全明确
5. **多个有效H2的处理**：同一回调内出现超过2个候选H2时的优先级规则，待补充
6. **仓位规模管理**：当前版本只定义仓位比例（50/50），未涉及账户风险百分比的具体规则
