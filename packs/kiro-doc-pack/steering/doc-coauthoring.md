# doc-coauthoring — 文档协同写作规范

> 本文件等价于 Claude 的 `/doc-coauthoring` skill，在本仓库 (priceAction) 内自动生效。
> 用于约束所有交易系统设计文档、研究笔记、复盘报告的写作风格与结构。

## 1. 文档分层与命名

```
设计文档.md            ← 当前生效版本（latest）
设计文档-v{X.Y}.md     ← 已发布的版本快照（不可修改）
REVIEW-v{X.Y}.md       ← 对应版本的评审/变更说明
价格行为AI.md           ← 自由形态的原始思考与对话记录（可乱）
```

- **设计文档.md** 永远指向当前最新生效版本。
- 升级时新建 `设计文档-v{X.Y}.md` 作为不可变快照，并写一份对应的 `REVIEW`。
- 重大版本（v1→v2）必须有 REVIEW；小版本（v2.0→v2.1）可只在 CHANGELOG 段落里写。

## 2. 结构标准（任何"系统设计文档"必须包含的章节）

1. **元信息**：版本、日期、品种、周期、作者
2. **系统目标与边界**：能做什么 / 不做什么 / 硬性约束
3. **决策流程总览**：编号化的 N 步序列（流程图 + 文字）
4. **每步详细规则**：定义、条件、阈值、失效条件
5. **核心参数一览**：表格列出全部可调参数及默认值
6. **风险预算与仓位管理**：账户级、日级、单笔级
7. **执行假设**：滑点、延迟、部分成交、订单类型
8. **已知缺口与迭代方向**：v 后续版本要补的事项
9. **附录**：变量字典、术语表、回测计划

## 3. 写作约定

### 3.1 K 线引用

- `bar[i]` 表示第 i 根**已收盘** K 线，`i=0` 为最新已收盘 bar。
- **永远不引用未收盘 bar**。所有信号在 bar close 时触发。
- 显式标注："使用已收盘 K 线"。

### 3.2 阈值

- 距离类阈值统一用 **ATR14 倍数** 表达，避免硬编码点数。
- 比例类阈值显式写百分比 + 公式：`Pullback_Ratio < 0.50 (50%)`。
- 阈值默认值必须可在"核心参数一览"表中查到。

### 3.3 条件块

用伪代码块（` ```plain `）表达条件，禁止用自然语言堆砌：

```plain
Condition_Name =
    sub_condition_A
    AND sub_condition_B
    AND NOT sub_condition_C
```

每个条件必须有：① 命名 ② 显式 AND/OR ③ 至少一行注释解释意图。

### 3.4 失效条件

每一步都要明确写出 **"若…则系统返回等待状态 / 取消挂单"**，不要让规则只有"通过"路径。

## 4. 变量命名

- 业务概念用 **PascalCase 或 Snake_Case 英文**：`Leg_High`, `Pullback_Ratio`, `Strong_Bull_Bar`。
- 中文只用于注释和说明，**不进入代码标识符**。
- 同一概念跨文档使用同一命名（参见变量字典附录）。

## 5. 评审要点（自检 checklist）

发布前自查：

- [ ] 每个条件块都有失效路径
- [ ] 每个阈值都有出处（参数表 / 注释 / 引用）
- [ ] 流程总览的步数与详细章节数一致
- [ ] 所有时间窗口（"近 N 根"）都明确写明 N
- [ ] 任何"可能"、"大概"、"通常"等模糊措辞都被替换为可计算条件
- [ ] 风险预算章节存在且有具体数字
- [ ] 执行假设（滑点等）章节存在
